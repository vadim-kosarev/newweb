Идея в следующем -- сыпать в базу данных разнотипные данные из совершенно различных событий
- процесс запущен
- тест пройден/не пройден
- получено письмо 
и т.д.

Эти данные должны сохраняться в БД. интерфейс должен быть максимально упрощенным.
Должна быть возможность легко строить запросы, раскрашенные таблицы по запросам и графики (какбе Excel).


===============================

Таблицы с префиксом sys_ -- системные, создаются во время установки ПО, не удаляются
Таблицы с префиксом data_ -- хранение данных. Могут удаляться для очистки базы. Создаются автоматически во время работы.


===============================
Добавление данных в базу данных:

genStaterAddEvent.cmd event=processStarted d_date_startDate=123338282828 \
    d_ext_host=BREEZE7 d_int_pid=99943 d_str_path=C:\WINDOWS\notepad.exe \
    tags=RTS,Test,Beta,3.04,PreRelease



event -- имя евента. Для каждого нового уникального имени эвента создается отдельная таблица
 data_events_processStarted

d_date_startDate -- поле данных. Для каждого такого поля создается столбец в таблице (??? автоматом или при создании таблицы)
 d_ == данные
 _date_ = тип данных - Дата
 _startDate = имя данных

d_ext_host=BREEZE7 -- поле данных. 
 d_ == данные
 _ext_ == означает, что для этих данных должна быть создана отдельная таблица (значения будут повторяться) и в исходную таблицу
  записывается только ID (внешний ключ)
 _host == имя данных. имя таблицы будет data_ext_host и будет содержать два поля id, data. При наличии строчки в таблице
   оттуда просто возьмется id, при отстутсвии в таблицу запишется строка [ "1", "BREEZE7" ]

d_int_pid -- поле данных. INT, записывается полем в data_events_processStarted

d_str_path -- поле данных. VARCHAR, записывается полем в data_events_processStarted

---

tags -- список тегов для записи... до 10 штук. Значения тегов хранятся в таблице
  sys_tags ( аналогично _ext_ данным )
  в таблице data_events_processStarted создается 10 полей с именами tag0, tag1, tag2, ..., tag9
